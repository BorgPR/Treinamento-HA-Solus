Trilha de Aprendizagem HA

Esta arquitetura foi desenhada para tolerar falhas de hardware e software através de múltiplos níveis de redundância. Estude os tópicos abaixo antes da avaliação.

1. Rede e VIP

O Keepalived garante que o IP 192.168.1.100 sempre responda, mesmo que o primeiro servidor queime. O VIP "pula" para o próximo nó em menos de 1 segundo.

2. Storage Distribuído

Usamos GlusterFS em modo réplica 3. Cada arquivo gravado é enviado simultaneamente para 3 discos físicos diferentes. Sincronismo 100% garantido.

3. Orquestração Swarm

O Docker Swarm mantém o número desejado de réplicas. Se um serviço cair, ele o recria em outro nó automaticamente.

4. Banco de Dados

A camada de dados Oracle é isolada da camada web para garantir que IOPS de escrita não afetem a latência de entrega de conteúdo estático.
