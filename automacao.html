Automação de Infraestrutura Solus

Utilize nossos scripts validados para preparar, configurar e colocar em produção toda a stack Solus de forma automática.

Atenção: O script abaixo deve ser executado em um servidor com Linux (Debian, Ubuntu ou derivados) que já possua o Docker instalado. O script verificará as dependências e inicializará o Swarm automaticamente se necessário.

Guia de Execução Rápida

1

Acesso Root

O script exige privilégios de root ou sudo para criar usuários e gerenciar o Docker.

2

Salvar Script

Salve o código como setup-solus.sh e dê permissão: chmod +x setup-solus.sh

3

Execução

Rode ./setup-solus.sh. O script é inteligente e verificará o estado do Swarm por você.

setup-solus.sh
#!/bin/bash

# --- VERIFICAÇÃO DE AMBIENTE ---
if [ "$EUID" -ne 0 ]; then 
  echo "Execute como root!"; exit 1
fi

# --- SETUP SWARM ---
SWARM_STATE=$(docker info --format '{{.Swarm.LocalNodeState}}')
if [ "$SWARM_STATE" != "active" ]; then
    docker swarm init
fi

# --- ESTRUTURA E DEPLOY ---
main() {
    criar_estrutura
    preencher_arquivos
    
    chown -R solus:solus /docker
    docker stack deploy -c nginx.yml proxy
    docker stack deploy -c web.yml solus
}

main

O que este script faz?

  • Cria usuário e grupo solus com UID/GID 1000.
  • Cria a rede Docker Overway solus-prd para comunicação entre containers.
  • Gera toda a árvore de diretórios em /docker/ de forma padronizada.
  • Cria arquivos de configuração base (Nginx, Apache, PHP, APIs).
  • Aplica permissões rígidas de segurança (especialmente para certificados SSL).
  • Realiza o Deploy Automático das stacks Docker.
